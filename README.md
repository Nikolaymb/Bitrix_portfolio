# МОЙ БОЕВОЙ НАБОР: BITRIX CORE API

**Привет!** Это коллекция из 4-х проектов, которые я собрал, чтобы показать свои навыки в работе с 1С-Битрикс. Здесь нет лишних библиотек и воды — только чистый, рабочий код.

Фокус сделан на **классическом ядре Bitrix (Core API)**, то есть на том, что реально используется в 80% проектов. Считай, это моя база.

## ЧТО ТУТ ПОКАЗАНО И ЗАЧЕМ

Каждый проект решает конкретную задачу, за которую обычно платит заказчик.

| Проект | Что это | Ключевой навык (для собеса) |

| **1. Feedback Form** | Компонент для приёма заявок | Работа с **CIBlockElement** и **bitrix\_sessid()**. |
| **2. Custom Property** | JSON-поле для элемента ИБ | **Расширение ядра** через события `OnIBlockPropertyBuildList`. |
| **3. Cleanup Agent** | Фоновая очистка старых данных | **Системное обслуживание** и работа с **CAgent**. |
| **4. Module Skeleton** | Шаблон для модуля | Понимание **архитектуры** и методов **DoInstall/DoUninstall**. |

---

## МОИ СКИЛЛЫ, КОТОРЫЕ ТУТ ВИДНЫ


* **Классическое Core API (C-функции):** Уверенно работаю с \`CIBlockElement\`, \`CModule\`, \`CAgent\`. Я знаю, где нужно использовать D7, но могу быстро писать код и на "классике".
* **Событийная модель:** Умею вмешиваться в логику ядра (создание своих типов свойств, обработчиков).
* **Архитектура:** Понимаю, как собирать код в правильную структуру (\`/local/components/\`, \`/local/php_interface/\`, \`/local/modules/\`), чтобы его было легко поддерживать.
* **Безопасность:** Использую \`bitrix\_sessid()\` и базовую валидацию, понимаю, как работает CAPTCHA.

## КАК ЗАПУСТИТЬ У СЕБЯ

Просто возьми папку проекта, который тебе нужен, и закинь ее содержимое в `/local/` своего тестового сайта Bitrix.
